# Канвас 6. Модель рва: драйверы + формула

Главный выход. Показывает, как бизнес, просто работая, сам наращивает порог копирования.

## Шаг 1. Слои защиты — что углубляется со временем
Отметь, какие слои в твоём бизнесе **растут** со временем. В модель идут только растущие. Софт обычно ✗ (копируется за квартал = «комик»); ров — в слоях, которые «смешны собой» (= «комедийный актёр»).

| Слой | Есть у нас? | Углубляется со временем? |
|---|---|---|
| Софт / функциональность | | ✗ обычно нет |
| Сетевой эффект | | ✓ при single-homing |
| Клиентская база / история | | ✓ |
| Данные с feedback-loop | | ✓ |
| Партнёры / каналы / дефолт | | ✓ |
| Процессы / операционная сложность | | ✓ |
| Бренд (образ ценности в голове клиента) | | ✓ медленно |
| Контракты / регуляторика / плечо | | ✓ если эксклюзивны |

## Шаг 3. Формула
> **Порог копирования(t) = C + Σ wᵢ · Драйверᵢ(t)**
> C — копируется за квартал (софт/фичи, ≈const). Драйверᵢ(t) растёт от работы на рынке. wᵢ — вес.

Запиши свою формулу словами:
- C (что у вас копируемо ≈ const): 
- Драйвер 1 растёт от: ___ → поднимает порог через: ___
- Драйвер 2 растёт от: ___ → поднимает порог через: ___
- Как драйверы усиливают друг друга (петля): ___

## Шаг 4. Критерий
- [ ] **dПорог/dt > 0 и не убывает** — каждый месяц работы поднимает порог быстрее, чем дешевеет копирование.
- Если порог не растёт от работы → это не ров, а временная фора.

> Пример (Авито): C = объявления (мелкий слой) + сетевой эффект(t) → база(t) → партнёры(t) → бренд(t), каждый питается работой рынка → защищённее со временем.
> Контрпример (прайс-агрегатор): ставка на C (умный софт), драйверы не накопились → съели.
