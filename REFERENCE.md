# Справочник: типология рвов, метрики, фейки

*Словарь, из которого собираются гипотезы на Этапе 1 и проверяются на Этапах 2–5.*

## Сквозная рамка

Преимущество в среднем **mean-reverting**: со временем оно тает, и длительность форы сокращается. Persistence реальна, но это *распределение*, не гарантия. Поэтому стартуем с презумпции «время против нас», и модель обязана показать механизм, по которому разрыв **растёт**, а не схлопывается.

## Типология (Benefit + Barrier)

Тип рва = из чего сделан *барьер*. Три класса: спросовые (барьер у клиента), предложенческие (барьер у компании), системные/стратегические (барьер в позиции и связке слоёв).

| Тип | Класс | Механизм барьера | Сила в эпоху ИИ |
|---|---|---|---|
| Сетевой эффект | спросовой | ценность растёт с числом юзеров | ★★★ — **только** single-homing + плотность + не-asymptotic; иначе слаб |
| Издержки переключения / lock-in | спросовой | клиенту дорого/больно уходить | ★★★ доказан (Farrell–Klemperer, Williamson) |
| Бренд | спросовой | платят премию «вслепую» | ★★ доказан, но **зреет 4–11 лет** → за полдня не подтвердить |
| Экономия масштаба | предложенческий | ниже себестоимость на юнит | ★ при высоком MES; для софта труден |
| Cornered resource | предложенческий | эксклюзивный доступ к активу | ★★★ редкие/регулируемые данные, талант, IP |
| Процессная сила | предложенческий | путь демо→прод не скопировать | ★★ накопленная сложность |
| Feedback-loop данные | предложенческий | продукт улучшается от использования | ★★ (в отличие от generic) |
| Дистрибуция / канал | системный | дефолт, эмбеддинг, своя аудитория | ★★★ часто главный |
| Counter-positioning | стратегический | инкумбент не повторит без каннибализации | ★★ силён на старте |
| Innovation stack | системный | слои взаимно усиливаются | ★★★ прочнейший, труден в сборке |
| Endogenous sunk costs | предложенческий | эскалация R&D/рекламы держит концентрацию | ★★★ «жёсткий» (Sutton) |

## Фейки (отсекать сразу)

| Фейк | Почему не ров |
|---|---|
| Фича / функциональность | копируется за недели |
| Generic-данные по объёму | устаревают, дешевеют, копируются (a16z сами опровергли) |
| Модель как моат | коммодитизирована; живёт лишь на foundation-слое |
| Временная фора | сжалась до «нескольких промптов и интерфейса» |
| «У нас network effects» по умолчанию | Uber/Lyft NE слабы: multihoming + asymptotic |
| Dynamic capabilities | тавтологично, нефальсифицируемо, пост-хок |

> Скорость — не ров, но право его построить. Данные сами по себе — слабый ров; работают только редкие / проприетарные / с feedback-loop.

## Метрики

**Запаздывающие (доказательство, видны через годы):** ROIC−WACC spread, persistence of economic profit, pricing power.

**Ранние (leading) — что снимаем в проверке.** Это **корреляты, не суррогаты** ROIC: рост метрики ≠ ров будет; повод копать, не вердикт.

| Тип рва | Ранний сигнал | Быстрая проверка |
|---|---|---|
| Switching / lock-in | retention на плато (не в ноль); NDR > 100% | price-increase test; switching-интервью |
| Сетевой эффект | растёт органика/рефералы; k > 0 | referral probe (внешняя выборка!) |
| Бренд | inbound/waitlist без платного трафика | ⚠ Sean Ellis 40% — self-selection bias |
| Дистрибуция | CAC падает при том же канале | fake-door / channel микро-тест |
| Cornered resource | конкурент не получит тот же доступ | red-team «достань ресурс за квартал» |
| Процессная сила | путь демо→прод у конкурента в 10–100× | red-team «скопируй надёжность, не демо» |

**Правило валидности:** поведенческий сигнал считается только во **внешнем холодном канале** с выборкой, посчитанной по мощности (n, при котором доверительный интервал не накрывает порог). Сигнал «на коллегах» = инсайт, не сигнал.
