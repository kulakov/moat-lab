# Metric-Mapper — ранняя метрика + порог — Этап 5

**Драйвер модели → ранняя метрика (производная накопления) + порог «сигнал» / «опровержение», помеченная «коррелят, не суррогат».**

## Что делает
Берёт один драйвер причинной модели и предлагает раннюю метрику — производную накопления на пользователя/время, — с порогами сигнала и опровержения. Жёстко метит метрику как коррелят (не доказательство рва) и делает опровержение адресным.

## Вход
- **Драйвер модели** — один накапливаемый механизм из канваса Этапа 4 (например: «глубина настройки workflow на клиента растёт с каждым месяцем»).
- Тип рва, к которому драйвер привязан (см. REFERENCE.md — switching, feedback-loop, дистрибуция и т.д.).
- Окно наблюдения и стадия (pre-product / post-revenue).

## Промпт-цепочка
1. **Назови, ЧТО должно накапливаться** по этому драйверу (объект накопления: настройки, связи, данные с feedback-loop, эмбеддинг в процесс клиента).
2. **Выбери раннюю метрику как ПРОИЗВОДНУЮ накопления** — не уровень, а скорость прироста на пользователя/время. Не «retention», а «как меняется retention-плато от когорты к когорте»; не «число интеграций», а «прирост глубины интеграции на клиента в месяц».
3. **Сверься с REFERENCE.md** — у типа рва есть свой ранний сигнал и быстрая проверка; не выдумывай метрику, если в справочнике уже есть подходящая.
4. **Задай порог «сигнал»** — какая динамика производной = «ров достижим, копать дальше» (НЕ «ров построен»).
5. **Задай порог «опровержение» АДРЕСНО.** Формула: «драйвер D этим способом S в окне W не накапливается». Никакого «рва нет вообще». Перед опровержением — чек отсечения альтернатив (канал / исполнение / стадия).
6. **Поставь штамп «коррелят, не суррогат».** Явно: рост этой метрики коррелирует с движением к рву, но не равен построенному рву; запаздывающее доказательство (ROIC−WACC, pricing power) видно только годами.

## Формат выхода
```
ИИ-черновик ранней метрики (проверяет команда)

Драйвер
  «Глубина настройки workflow под клиента растёт со временем»  (тип: switching/lock-in)

Что накапливается
  Кол-во и незаменимость кастомных конфигов, встроенных в рабочий процесс клиента.

Ранняя метрика — ПРОИЗВОДНАЯ накопления
  Δ(глубина интеграции) на активного клиента в месяц,
  + сдвиг retention-плато по когортам (плато выше у более старых когорт?).
  (не уровень, а скорость прироста и тренд по когортам)

Пороги
  СИГНАЛ (копать дальше): производная > 0 и плато по когортам РАСТЁТ
    → ров достижим. НЕ значит «ров построен».
  ОПРОВЕРЖЕНИЕ (адресное): за окно W=3 мес производная ≈ 0 И плато по
    когортам не растёт → «драйвер настройки workflow этим способом в этом
    окне не накапливается». НЕ «рва нет».
  Перед опровержением отсечь: канал / исполнение / стадия.

⚠ Коррелят, не суррогат
  Рост метрики ≠ ров построен. Это leading-коррелят ROIC, не доказательство.
  Доказательство (ROIC−WACC spread, pricing power) видно только через годы.

Уверенность: средняя (метрика правдоподобна для типа; валидность порога —
  за командой и фактическими данными).
```

## Gate
- **ОБЯЗАТЕЛЬНО помечай метрику «коррелят, не суррогат»** — рост метрики не равен построенному рву; запаздывающее доказательство видно годами.
- **Метрика — производная накопления**, не статичный уровень; если предлагается уровень — переформулируй в скорость прироста / тренд по когортам.
- **Опровержение формулируй АДРЕСНО** — «этот драйвер этим способом в этом окне не накапливается», никогда «рва нет вообще»; перед выводом отсеки альтернативы (канал / исполнение / стадия).
- **Презумпция истощения** — по умолчанию «время против нас»; метрика должна ловить именно перебивающий механизм (разрыв растёт), а не общий рост.
- Свой вклад помечай «ИИ-черновик/аргумент». Вердикт НЕ выносишь — его выносит команда/фасилитатор.
- Faithfulness · traceability · uncertainty.
