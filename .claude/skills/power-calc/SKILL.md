# Power-Calc — минимальный n внешнего теста — Этап 4

**Порог + ожидаемый эффект → минимальный размер выборки n, при котором доверительный интервал не накрывает порог.**

## Что делает
Берёт порог конверсии/сигнала и ожидаемый реальный эффект, считает минимальный n холодного внешнего теста, при котором CI вокруг наблюдаемой доли не задевает порог. Объясняет на пальцах, почему маленькая выборка — шум, и фиксирует: без достаточного n вывод «сигнал есть» запрещён.

## Вход
- **Порог** — какая конверсия/доля считается сигналом (например, «≥10% от холодного трафика оставляют заявку»).
- **Ожидаемый эффект** — какую долю реально надеемся увидеть (например, «думаем будет ~18%»).
- Канал теста (нужен для напоминания: только внешний холодный, не «на коллегах»).
- Опционально: доступный объём трафика/контактов — чтобы честно сказать, добиваем ли мы нужный n.

## Промпт-цепочка
1. **Зафиксируй порог p₀ и ожидаемую долю p₁.** Если эффект задан как «лучше порога» без числа — попроси число; без p₁ оценка n невозможна, помечай «неизвестно».
2. **Проверь зазор |p₁ − p₀|.** Чем он меньше, тем больше n. Если зазор < 3 п.п. — предупреди: такой тест за окно почти нереалистичен, нужны сотни-тысячи наблюдений.
3. **Прикинь n правилом большого пальца (без тяжёлой статистики):**
   - Полуширина 95%-CI для доли ≈ `1.96 · √(p·(1−p)/n)`. Хотим, чтобы она была **меньше зазора** до порога.
   - Рабочая формула прикидки: `n ≈ 4 · p·(1−p) / d²`, где `d = |p₁ − p₀|` (доля, не проценты), коэффициент 4 ≈ 1.96².
   - Считай при `p = p₁` (наблюдаемая) и при `p = p₀` (граничная) — бери больший n, это консервативно.
4. **Округли вверх и добавь запас 15–20%** на отказы/брак трафика. Дай диапазон, не одно число.
5. **Покажи «что будет при малом n»:** при n=12 полуширина CI ≈ `1.96·√(0.25/12)` ≈ ±28 п.п. — то есть наблюдённые 18% означают «где-то от −10% до +46%», CI накрывает любой разумный порог → **сигнала нет, есть шум**.
6. **Сформулируй gate-строку:** «вывод "сигнал есть" разрешён только при n ≥ N и наблюдаемой доле, чья нижняя граница CI выше порога».
7. **Напомни про pre-registration:** порог и n фиксируются ДО сбора данных. Pre-registration убирает подгонку порога под результат, но НЕ лечит недостаток мощности — маленький n остаётся мусором даже при честной регистрации.

## Формат выхода
```
ИИ-черновик расчёта мощности (проверяет команда)

Вход
  Порог p₀:        10%
  Ожидаем p₁:      18%
  Зазор d:         8 п.п.
  Канал:           холодный outreach в LinkedIn (внешний — ок)

Минимальный n
  по правилу n ≈ 4·p(1−p)/d²
  при p=0.18:  4·0.18·0.82 / 0.08²  ≈ 92
  при p=0.10:  4·0.10·0.90 / 0.08²  ≈ 56
  берём больший → ~92
  + запас 15–20% на брак → планировать 105–110 откликов

«На пальцах»
  При n=12 CI ≈ ±28 п.п. — наблюдённые 18% это диапазон −10…+46%.
  Такой интервал накрывает порог 10% → отличить сигнал от шума НЕЛЬЗЯ.
  Чтобы CI (±5 п.п.) не доставал до 10% от 18% — нужно ~90+.

Gate
  Вывод «сигнал есть» РАЗРЕШЁН только если: n ≥ 92 И нижняя граница 95%-CI
  наблюдаемой доли > 10%. Иначе — «данных недостаточно», не «нет рва».

Pre-registration
  Зафиксировать p₀=10%, n=92, канал и дату ДО первого отклика.
  Снимает подгонку порога, но не заменяет мощность.

Уверенность: средняя (нормальная аппроксимация; при p близко к 0/1 или n<30
  правило огрубляет — пометить как ориентир, не точную границу).
```

## Gate
- **Считай n так, чтобы доверительный интервал НЕ накрывал порог.** Если предложенный командой n этого не обеспечивает — покажи фактическую ширину CI и почему вывод будет недостоверным.
- **Без достаточного n вывод «сигнал есть» ЗАПРЕЩЁН.** Маленькая выборка → максимум «данных недостаточно», и опровержение тоже не выносится (нельзя сказать ни «есть», ни «нет»).
- **Pre-registration убирает подгонку, но не лечит мощность** — не выдавай зарегистрированный недомощный тест за валидный.
- **Только внешний холодный канал.** Тест «на коллегах/знакомых» в расчёт мощности не идёт — это инсайт, не сигнал.
- Свой вклад помечай «ИИ-черновик/аргумент». Вердикт НЕ выносишь — его выносит команда/фасилитатор.
- Faithfulness · traceability · uncertainty.
