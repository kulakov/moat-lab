# Teardown-prep — Этап 0

**Вход → Выход:** карта продукта → 3–5 структурно похожих компаний-победителей с реверсом механизма их барьера (каждый помечен публичный/непубличный, с источником).

## Что делает
Ищет компании, структурно похожие на продукт команды (тот же класс барьера, не та же тема), и реверсит, ЗА СЧЁТ ЧЕГО они стали неконкурируемыми. Цель — дать команде разобранные образцы реальных рвов до того, как они начнут формулировать свой.

## Вход
Заполненная карта продукта (`canvases/00-product-map.md`), особенно поля: однострочник, сегмент/ЛПР, статус-кво, что накапливается, эксклюзивный доступ, каналы.

## Промпт-цепочка
1. **Извлеки структурный профиль** из карты: какой класс барьера команда нащупывает (спросовой / предложенческий / системный — по REFERENCE.md), кто ЛПР, какой статус-кво вытесняется. Не тему («ещё один таск-трекер»), а структуру («B2B-инструмент, который застревает в рабочем процессе клиента»).
2. **Подбери 3–5 победителей того же структурного класса.** Победитель = время работало на него, разрыв с догоняющими рос. Разнообразь источник барьера, не клонируй один тип.
3. **Для каждого реверсни механизм барьера:** не «у них хороший продукт», а конкретно — за счёт чего догоняющему дорого/долго/больно повторить (накопленные данные с feedback-loop, cornered resource, switching-издержки, innovation stack — по типологии REFERENCE.md).
4. **Классифицируй каждый механизм публичный/непубличный.** Публичный = описан в кейсах/блогах/докладах → значит копируется, в зачёт НЕ идёт как ров. Непубличный = выводится только из наблюдения за поведением рынка, не лежит в открытом плейбуке.
5. **Поставь источник** на каждое утверждение (URL, кейс, отчёт) или пометь «вывод ИИ, без прямого источника — проверить».
6. **Отметь уверенность** по каждому победителю (высокая/средняя/низкая) и где это догадка о механизме, а не факт.

## Формат выхода
```
## Структурный профиль команды (ИИ-черновик)
Класс барьера: предложенческий (feedback-loop данные)
ЛПР: руководитель отдела поддержки
Статус-кво вытесняется: ручные макросы в Zendesk

## Победители-аналоги

### 1. [Компания] — ★★ уверенность средняя
Механизм барьера: каждый разрешённый тикет дообучает их роутинг → точность растёт с объёмом, догоняющий стартует с нуля.
Тип (REFERENCE): feedback-loop данные.
Публичный/НЕПУБЛИЧНЫЙ: НЕПУБЛИЧНЫЙ — в зачёт (в открытых кейсах подаётся как «AI», механизм накопления не раскрыт).
Источник: [URL] / «вывод ИИ из поведения рынка — проверить».
Чему учит команду: их собственный feedback-loop сейчас generic — копируется.

### 2. ...
```
В конце — строка: **«В зачёт идут только победители с непубличным механизмом. Публичные — справочно, как то, что уже скопировано».**

## Gate
- ЗАСЧИТАН ТОЛЬКО НЕПУБЛИЧНЫЙ механизм. Публичный механизм помечай явно и выноси как «это копируют» — он не образец рва.
- Каждый победитель и каждый механизм — с источником или явной пометкой «вывод ИИ, проверить».
- Свой вклад помечай «ИИ-черновик/аргумент». Вердикт НЕ выносишь — его выносит команда/фасилитатор.
- Faithfulness (не выдумывай компании и механизмы; чего не знаешь — «неизвестно») · traceability (источник на утверждение) · uncertainty (показывай уверенность по каждому аналогу).
