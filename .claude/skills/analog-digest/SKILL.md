# Analog-digest — Этап 1

**Вход → Выход:** запрос на конкретный аналог → углублённый разбор механизма барьера победителя (непубличный механизм или явное «это не барьер»; источники).

## Что делает
Глубоко разбирает ОДНУ названную компанию: за счёт чего у неё растёт разрыв с догоняющими — или честно показывает, что заявленный «ров» на деле фича/фора/generic-данные. В отличие от `teardown-prep` (оптом 3–5 на Этапе 0), это адресный дайвинг по запросу команды на Этапе 1.

## Вход
Имя конкретной компании-аналога от команды + (желательно) гипотеза команды, какой у неё барьер.

## Промпт-цепочка
1. **Зафиксируй заявку:** какую компанию и какой её предполагаемый барьер просят разобрать.
2. **Разбери механизм по слоям:** что накапливается со временем, почему догоняющему дорого/долго/больно это повторить, растёт ли разрыв или это статичная фора. Привяжи к типу из REFERENCE.md.
3. **Прогони тест копирования:** можно ли это повторить за квартал, имея бюджет? Если да — это НЕ барьер, скажи прямо (вероятно фича / generic-данные по объёму / модель / временная фора — фейки §1.2 REFERENCE).
4. **Классифицируй механизм публичный/непубличный.** Публичный (лежит в кейсах/докладах/плейбуках) → копируется, в зачёт как ров не идёт. В зачёт — только непубличный, выводимый из поведения рынка.
5. **Источник на каждое утверждение** (URL/кейс/отчёт) либо пометка «вывод ИИ, проверить». Не выдумывай внутренние факты компании.
6. **Вывод для команды:** что из механизма этого аналога применимо к их гипотезе, а что — ловушка (выглядит как ров, но копируется).

## Формат выхода
```
## Заявка
Компания: [X]. Предполагаемый барьер команды: «у них больше данных».

## Разбор механизма (ИИ-черновик) — уверенность средняя
Что накапливается: размеченные сценарии с feedback-loop, не сырой объём.
Почему дорого повторить: догоняющий стартует с холодным loop, точность ниже,
клиенты не дают данные плохому продукту → петля не запускается.
Тип (REFERENCE): feedback-loop данные (★★).
Тест копирования за квартал: НЕ проходит — петлю за квартал не накопить.

## Публичный / НЕПУБЛИЧНЫЙ
НЕПУБЛИЧНЫЙ механизм → в зачёт. (В блогах подаётся как «AI-first»,
устройство петли не раскрыто.)
Источник: [URL кейса] / «вывод ИИ из поведения рынка — проверить».

## ИЛИ: ЭТО НЕ БАРЬЕР
Если бы накапливался generic-объём — копируется за квартал, фейк §1.2.

## Для гипотезы команды
Применимо: их собственный loop сейчас generic — перенести акцент на разметку.
Ловушка: «больше данных» само по себе ≠ ров.
```

## Gate
- В зачёт — только непубличный механизм; иначе явно «это не барьер» (тест копирования за квартал). Источник на каждое утверждение.
- Свой вклад помечай «ИИ-черновик/аргумент». Вердикт НЕ выносишь — его выносит команда/фасилитатор.
- Faithfulness (не выдумывай внутренние факты компании; неизвестное — «неизвестно») · traceability (источник/ссылка на тип REFERENCE на каждое утверждение) · uncertainty (уверенность в разборе видна).
