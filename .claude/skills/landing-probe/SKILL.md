# Landing-Probe — Этап 4

**Вход:** выбранная гипотеза + аудитория → **Выход:** рабочий лендинг или outreach-цепочка за минуты под внешний холодный тест.

## Что делает
Собирает один тестопригодный артефакт — лендинг (fake-door) или цепочку холодного outreach — ровно под проверяемую гипотезу. Цель не «красиво», а «можно запустить во внешний канал и измерить отклик». Решение, запускать ли и что считать сигналом, — за командой.

## Вход
- Выбранная на Этапе 3 гипотеза + критерий теста, названный ДО выбора.
- Аудитория: реальный сегмент/ЛПР (поле карты 2), внешний холодный канал (не коллеги).
- Развилка стадии (CLAUDE.md): **pre-product** → fake-door на тест **спроса**; **post-revenue** → тест на реальных метриках.
- Что именно меряем (от power-calc, если уже есть): порог отклика + минимальный n.

## Промпт-цепочка
1. **Зафиксируй, что тестируем** — одну гипотезу и один измеримый отклик (клик/заявка/ответ на письмо). Не больше одного на артефакт.
2. **Выбери формат под канал:** холодная аудитория без трафика → outreach-цепочка; есть способ привести холодный трафик → лендинг fake-door.
3. **Текст строго под гипотезу** — benefit и барьер из формулировки, на языке ЛПР (свяжи с customer-oracle). Никаких обещаний, которых продукт не несёт.
4. **Встрой точку замера** — CTA = ровно то событие, что считается сигналом (предзаказ, demo-request, ответ). Без неё артефакт не тестопригоден.
5. **Pre-registration отклика** — что считаем «сигнал» vs «опровержение» ДО запуска (порог + n от power-calc). Внутризальный отклик = «инсайт, не сигнал».
6. **Собери артефакт:**
   - лендинг — самодостаточный HTML (своя вёрстка, **без чужих лого/брендов**, без скриншотов несуществующего продукта сверх честного fake-door);
   - outreach — 2–3 письма (cold + follow-up) с одним CTA.
7. **Чек на пустышку** — артефакт обещает ровно проверяемое, точку замера видно, чужих активов нет. Если не проходит — переделай, не отдавай.

## Формат выхода
```
ИИ-черновик (landing-probe). Запуск и трактовку решает команда.

Тестируем: <гипотеза> | Отклик-сигнал: <событие> | Канал: <внешний холодный>
Стадия: pre-product (fake-door) / post-revenue
Pre-registration: сигнал ≥ <порог> при n ≥ <от power-calc>; ниже — опровержение (адресное)

АРТЕФАКТ
— если лендинг: <самодостаточный HTML-файл, путь>, секции: hero(benefit+барьер) · как работает · CTA(=точка замера)
— если outreach:
  Письмо 1 (cold): <тема + тело, 1 CTA>
  Письмо 2 (follow-up): <…>

Чек тестопригодности: точка замера ✓ · без чужих лого ✓ · обещает только проверяемое ✓
Открытые вопросы: <чего не хватает для запуска>.
```

## Gate
- **Точно по выбранному решению** — артефакт тестирует ровно ту гипотезу, что выбрана на Этапе 3, и ровно один отклик.
- **Без «красивой пустышки»** — есть точка замера и pre-registration сигнала; артефакт реально запускается во внешний холодный канал.
- **Без чужих лого/брендов** и без скриншотов несуществующего продукта сверх честного fake-door.
- Внутризальный отклик не считается — только внешний холодный канал с n от power-calc.
- Свой вклад помечай «ИИ-черновик/аргумент». Вердикт НЕ выносишь — его выносит команда/фасилитатор.
- Faithfulness · traceability · uncertainty: текст не обещает того, чего продукт не несёт; чего не знаешь про аудиторию — спроси, не выдумывай.
